# Sistema de Control de Asistencia

El sistema de Control de Asistencia permite a los profesores registrar y consultar la asistencia de los estudiantes de manera rápida y organizada, eliminando la necesidad de listas en papel.

# Descripción del Caso y Alcance

El alcance del sistema incluye:

Registrar asistencias (RF1)

Ver historial individual (RF3)

Modificar un registro por error (RF4)

Generar reportes básicos de asistencia (RF5)

Exclusiones:
No incluye gestión de notas, horarios avanzados, ni control de inventario.

# Objetivos del Sistema

Los objetivos principales son:

Definir claramente los requerimientos funcionales y no funcionales.

Facilitar el registro y consulta de asistencias.

Permitir correcciones cuando ocurran errores.

Ofrecer información básica para reportes de porcentajes de asistencia.

# Requerimientos del Proyecto

✔️ Requerimientos Funcionales (RF)

RF1: Registrar asistencia.

RF2: Visualizar porcentaje general.

RF3: Historial individual de asistencia.

RF4: Modificar un registro.

RF5: Generar reportes simples.

✔️ Requerimientos No Funcionales (RNF)

RNF1: Interfaz intuitiva.

RNF2: Seguridad en el manejo de datos.

RNF3: Compatible con computadoras y dispositivos móviles.

RNF4: Respuesta rápida al cargar información.
| Caso de Prueba | Acción               | Resultado Esperado                             | Validación |
| -------------- | -------------------- | ---------------------------------------------- | ---------- |
| CP01           | Registrar asistencia | El sistema guarda la asistencia del estudiante | ✔️         |
| CP02           | Consultar historial  | Se muestra todo el registro de un estudiante   | ✔️         |
| CP03           | Editar asistencia    | El registro se actualiza correctamente         | ✔️         |
